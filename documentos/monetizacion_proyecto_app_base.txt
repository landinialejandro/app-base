<<SECTION: OBJETIVO>>

OBJETIVO

Este documento define el modelo general de monetización de app-base.

Su función es:

definir cómo el proyecto genera ingresos
ordenar líneas comerciales vigentes
establecer planes de membresía
formalizar extras y upgrades
definir modalidad enterprise
alinear evolución comercial con la visión del producto
dejar explícitos criterios de confianza comercial y portabilidad

No define:

detalle técnico de módulos
implementación de pricing en código
campañas de marketing tácticas
costos contables internos
copy final de landing o publicidad

<<SECTION: PROPUESTA DE VALOR>>
SECTION_VERSION: 00001

PROPUESTA DE VALOR

app-base monetiza resolviendo dos necesidades principales del mercado:

ordenar la operación interna de las empresas
habilitar nuevas oportunidades comerciales y crecimiento digital

El cliente no compra código.

El cliente compra:

orden
continuidad operativa
control
claridad para trabajar
una base profesional para crecer
capacidad de ampliar el negocio sin perder coherencia

La propuesta de valor del proyecto debe leerse como una base de trabajo preparada para elevar la forma en que opera una empresa.

No se presenta solo como software.

Se presenta como un entorno propio, claro, sólido y preparado para acompañar crecimiento real.

<<END SECTION>>

<<SECTION: POSICIONAMIENTO COMERCIAL>>
SECTION_VERSION: 00001

POSICIONAMIENTO COMERCIAL

app-base se posiciona como una plataforma de gestión y crecimiento para empresas que necesitan ordenar su operación, trabajar con más claridad y sostener una evolución más profesional sin perder simplicidad.

No debe posicionarse como una suma de módulos sueltos.

Debe posicionarse como una infraestructura premium para empresas.

Conceptos centrales del posicionamiento vigente:

entorno propio
orden
continuidad
control
crecimiento con coherencia
base sólida para trabajar mejor

<<END SECTION>>

<<SECTION: LÍNEAS COMERCIALES>>
SECTION_VERSION: 00001

LÍNEAS COMERCIALES

El proyecto opera inicialmente en dos líneas comerciales complementarias:

MEMBRESÍA

Acceso al producto estándar en modalidad SaaS.

Incluye:

uso del sistema
hosting administrado
mantenimiento general
mejoras evolutivas comunes
soporte según plan

Esta línea comercial debe comunicarse como acceso a una infraestructura lista para trabajar y preparada para crecer.

IMPLEMENTACIÓN PARTICULAR

Modalidad orientada a clientes que requieren una solución exclusiva o personalizada.

Puede incluir:

instancia dedicada
servidor propio o administrado
branding propio
dominio propio
migración de datos
integraciones
ajustes funcionales
soporte preferente
capacitación inicial

Esta línea no debe leerse solo como plan más grande.

Debe leerse como modalidad comercial distinta, orientada a organizaciones que necesitan mayor exclusividad, control o acompañamiento.

USUARIOS ADICIONALES

Los planes estándar admiten ampliación mediante usuarios adicionales.

Valor vigente:

USD 5,5 por usuario adicional / mes

Reglas:

el usuario adicional resuelve necesidades intermedias de crecimiento
no reemplaza la lógica natural de upgrade entre planes
la estrategia comercial puede sugerir migración al plan superior cuando resulte más conveniente para el cliente

DOCUMENTS COMO CAPA AVANZADA

El módulo Documents se habilita desde Growth.

Criterio comercial vigente:

requiere mantenimiento continuo
puede demandar actualizaciones fiscales o normativas
implica mayor responsabilidad operativa y soporte
agrega valor claro en empresas con procesos más maduros

Su exclusión de Starter responde a segmentación estratégica y sostenibilidad del servicio.

No debe comunicarse como recorte arbitrario.

Debe comunicarse como una capa más exigente del producto, asociada a formalización operativa, mantenimiento especializado y evolución regulatoria.

<<SECTION: PORTABILIDAD Y CONFIANZA COMERCIAL>>
SECTION_VERSION: 00001

PORTABILIDAD Y CONFIANZA COMERCIAL

El vínculo comercial con el cliente debe sostenerse por valor entregado y no por dependencia forzada.

Regla general:

la información principal del cliente debe poder exportarse mediante un proceso de salida razonable, seguro y útil.

Criterios vigentes:

no corresponde entregar código fuente
no corresponde entregar configuraciones propietarias del sistema
no corresponde incluir usuarios internos como parte de la portabilidad estándar
no corresponde incluir catálogos internos cuya utilidad dependa de lógica propia de la plataforma

Sí corresponde contemplar la exportación de datos maestros y operativos principales del negocio cuando resulte aplicable.

Cuando sea posible, las referencias internas deben resolverse a texto legible y no quedar limitadas a ids técnicos.

Si una referencia no pudiera resolverse razonablemente, podrá conservarse el identificador técnico correspondiente.

<<END SECTION>>

<<SECTION: ENTREGA DE EXPORTACIONES Y BAJA>>
SECTION_VERSION: 00001

ENTREGA DE EXPORTACIONES Y BAJA

La salida del cliente no debe prometerse como entrega inmediata universal ni como descarga irrestricta sin control.

Debe existir un proceso formal y seguro.

Modalidades posibles según volumen:

exportación estándar mediante archivos descargables
exportación asistida para volúmenes altos
entrega temporal mediante enlace seguro con vigencia limitada

La doctrina comercial no debe atar esta capacidad a una herramienta puntual.

Lo relevante es preservar:

seguridad
vigencia controlada
acceso por titular autorizado
portabilidad razonable de la información principal

<<END SECTION>>

<<SECTION: PRINCIPIOS DE PRICING>>
SECTION_VERSION: 00001

PRINCIPIOS DE PRICING

El pricing del proyecto debe sostenerse en criterios simples y entendibles.

Principios vigentes:

precio claro
progresión simple entre planes
upgrade natural
sin castigos artificiales
sostenibilidad operativa
alineación con valor entregado

La lectura económica principal no debe reducirse a precio por módulo.

Debe responder a:

capacidad operativa
cantidad de usuarios
nivel de soporte
complejidad de mantenimiento
exigencia funcional del servicio

Enterprise queda fuera de la lógica estándar de pricing público y se resuelve por propuesta específica.
