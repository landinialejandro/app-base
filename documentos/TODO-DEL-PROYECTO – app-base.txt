Objetivo

Este documento registra decisiones pendientes, mejoras futuras, profundizaciones postergadas y líneas de evolución del proyecto app-base.

Su finalidad es:

no perder ideas importantes

evitar profundizar demasiado pronto

separar lo urgente de lo evolutivo

mantener un backlog técnico/funcional vivo

permitir actualización continua a medida que surjan nuevas necesidades

Este documento debe crecer con el proyecto.

1. Regla de uso del archivo TODO

Este archivo debe registrar:

decisiones postergadas

mejoras de arquitectura no urgentes

refactorizaciones futuras

módulos futuros no prioritarios

validaciones avanzadas

ideas de escalabilidad

racionalizaciones del dominio

mejoras de UX o estructura

No deben agregarse aquí tareas ya cerradas o decisiones ya consolidadas en los documentos fijos.

12. Cómo actualizar este archivo

Cada vez que aparezca una idea, mejora o tema no prioritario, agregarlo bajo una sección existente o crear una nueva.

Formato sugerido para agregar nuevos TODO:

tema

motivo

urgencia: baja / media / alta

impacto estimado

dependencia con otros módulos

estado: pendiente / en análisis / resuelto

Ejemplo:

definir secuencia documental por tenant

motivo: necesaria para documents

urgencia: media

impacto: medio

dependencia: módulo documents

estado: pendiente

13. Regla final del TODO

Este archivo no debe bloquear el avance.

Su función es permitir que el proyecto avance con una base operativa, dejando registradas las profundizaciones futuras para tratarlas en el momento adecuado.
