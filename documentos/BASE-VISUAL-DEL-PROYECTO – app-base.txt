OBJETIVO

Definir una base visual simple y consistente para el sistema app-base que permita desarrollar módulos con una interfaz uniforme, clara y fácil de mantener.

ESTRUCTURA GENERAL DE PÁGINA

Todas las páginas internas del sistema deben seguir la siguiente estructura:

<x-page> <x-breadcrumb /> <x-page-header /> <x-card /> </x-page>

Esta estructura establece un patrón visual uniforme para todas las pantallas del sistema.

COMPONENTES DISPONIBLES

Componentes Blade reutilizables para la interfaz:

x-page
Contenedor principal de una pantalla.

x-breadcrumb
Ruta de navegación dentro del sistema.

x-page-header
Encabezado de la página con título y acciones principales.

x-card
Contenedor visual para contenido de la página.

REGLAS DE LISTADOS

Las vistas de tipo listado (index) deben seguir el patrón estándar de listas.

Estructura recomendada:

<x-page class="list-page"> <x-breadcrumb /> <x-page-header /> <x-card class="list-card"> <div class="table-wrap list-scroll"> <table class="table">

Clases CSS obligatorias para listados:

page.list-page
card.list-card
table-wrap.list-scroll

Características del patrón de listas:

la tabla se encuentra dentro de una card

el listado utiliza scroll interno

el encabezado de la tabla permanece visible

el breadcrumb y el título permanecen visibles

REGLAS DE FORMULARIOS

Los formularios deben reutilizar parciales.

Estructura recomendada:

_form.blade.php

Este archivo es utilizado por:

create.blade.php
edit.blade.php

Reglas generales:

reutilizar el mismo formulario para crear y editar

agrupar campos de forma clara

ubicar botones dentro de un bloque de acciones

ESTADOS VACÍOS

Cuando un listado no tenga registros, el mensaje debe mostrarse dentro de la card del listado.

TERMINOLOGÍA DE INTERFAZ

La interfaz del sistema debe utilizar términos orientados al usuario.

Equivalencias:

Tenant → Empresa
Party → Contacto

La nomenclatura técnica puede mantenerse en el código, pero la interfaz debe utilizar los términos funcionales.

MÓDULOS QUE DEFINEN EL PATRÓN

Los siguientes módulos ya utilizan esta base visual:

projects
parties (contactos)

Los nuevos módulos deben seguir el mismo patrón visual.

REGLA DE CONSISTENCIA

Todos los módulos nuevos deben utilizar:

la misma estructura de página

los mismos componentes Blade

el mismo patrón de listados

las mismas reglas de formularios

Esto garantiza coherencia visual en todo el sistema.
