LECCIONES VIGENTES

Ability sobre módulo inexistente puede fallar por inexistencia contractual y no por falta de permiso.

Dataset interno reusable y autorización efectiva son problemas distintos.

Una falla visual reusable puede deberse a hostPack, slot, type, shape o view desalineados y no a permisos.

Sin tenant activo en Tinker pueden aparecer falsos negativos.

No revisar solo controller principal: también tabs, modales, partials y superficies secundarias.

Un host visual no se convierte en dueño funcional de la capacidad invitada.

No toda necesidad repetida exige módulo nuevo.

Cambiar estados operativos requiere alinear Security, Gate, UI y ciclo de vida real.

Si una decisión importante no queda documentada, reaparece como discusión futura.

Si una entidad separa tipo base y relación operativa, la seguridad no debe seguir usando la dimensión anterior por compatibilidad.

Caso vigente:

parties.kind no debe arbitrar permisos.

party_roles.role debe arbitrar permisos mediante allowed_party_roles.

Si una opción sensible no aparece en UI, eso no alcanza como protección.

Caso vigente:

aunque el formulario de una party vinculada a membership activa no exponga de forma simple campos manipulables, debe probarse intento manipulado y bloqueo backend.

Las validaciones sensibles deben verificarse por persistencia final, no solo por mensaje visual.

Caso vigente:

tras intentar manipular kind, email y roles de una party vinculada a membership activa, debe verificarse que el registro siga intacto.

Lectura esperada:

kind: person
roles: employee

Los snippets para Project Lab deben respetar estrictamente el contrato del automatizador.

Error detectado:

usar encabezados no compatibles o mezclar FILE con TARGET puede hacer que el automatizador interprete incorrectamente el bloque como archivo completo.

Regla:

no mezclar formatos.

Usar FILE solo para archivo completo.

Usar TARGET solo para operación parcial.

Formatos correctos:

// FILE: ruta | Vn
// TARGET: ruta :: metodoExistente
// TARGET: ruta ++ metodoNuevo

No usar:

// FILE: ruta :: metodo

<?php // FILE: ruta | V1

Lección:

un formato ambiguo no debe tratarse como éxito operativo.

Debe corregirse el snippet antes de aplicar.

tenant_id debe respetar el tipo real de tenants.id.

Error detectado:

OperationalActivityLogger trató tenant_id como entero, pero tenants.id es UUID char(36).

Corrección:

conservar tenant_id como string.

Lección:

no asumir tipos de claves.

Validar esquema real antes de persistir datos transversales.

Event discovery puede registrar listeners automáticamente.

Error detectado:

RegisterOperationalActivity@handle ya estaba registrado por event discovery y además se registró manualmente en AppServiceProvider.

Resultado:

dos jobs
dos registros idénticos

Corrección:

eliminar el registro manual.

Lección:

si event:list ya muestra la asociación, no duplicar con Event::listen() manual.

php artisan tinker --execute puede fallar con use según el wrapper.

Error detectado:

bloques con use ... pueden parsearse mal dentro del wrapper de ejecución.

Alternativas:

usar clases con namespace completo
ejecutar el bloque en Project Lab según corresponda

Lección:

si el problema es el wrapper y no la lógica probada, ajustar la forma de ejecución antes de cambiar código funcional.

OperationalActivity no debe bloquear la operación principal.

Lección:

la trazabilidad operativa queued debe registrar si corresponde, pero un fallo del log no debe romper el flujo principal ya autorizado y ejecutado por el módulo dueño.
