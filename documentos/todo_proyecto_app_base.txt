vistas dedicadas para alta y edición

segmentación visual por kind
control de acceso mediante policies y acciones visibles condicionadas por permisos

El modelo de datos ya contempla soporte para:

clasificación funcional mediante kind
ordenamiento mediante sort_order
identificación de tipo de archivo mediante mime y extension
preparación para extensiones futuras como análisis, texto extraído y metadata adicional

Convenciones ya implementadas:

kind es la dimensión principal de organización visual del módulo

la extensión del archivo es un dato secundario y no define agrupación

el formulario de adjuntos permite definir tipo y descripción

la navegación del flujo se resuelve mediante retorno al recurso padre

las acciones de tabla son icon-only y respetan permisos reales

el acceso a archivos se resuelve mediante preview inline y download
